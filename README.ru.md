**plan-tango** — плагин для Claude Code: запускает ревью-цикл между Claude и Codex (gpt-5) одной командой.

Несколько раз ловил себя на том, что отправляю план от Claude в Codex руками. Замечания от GPT обычно полезные, и в одну итерацию это закрывается редко. Написал план, скопировал в Codex, дождался замечаний, перенёс обратно в Claude, попросил применить, отправил обновлённый план в Codex на следующий раунд, дождался новых замечаний, перенёс назад. И ещё раз. Между раундами ждёшь, периодически теряешь, в каком из терминалов сейчас актуальный вердикт и требуется действие.

plan-tango прогоняет этот цикл сам. Claude и Codex работают как пара, плагин передаёт контекст между ними и применяет правки в плане. По умолчанию — до 6 итераций, жёсткий потолок 12. Плагин остановится раньше, если не будет важных замечаний. Ты остаёшься в plan mode и можешь уйти заниматься другими делами; когда возникнет вопрос, требующий тебя, плагин остановится и спросит. Для срочных прогонов (или просто если есть свободные лимиты в Codex) — `--fast` (приоритетный режим Codex, примерно в 1.5 раза быстрее).

## Установка

Понадобится: Claude Code 2.x, Node.js 18+.

**1. Поставь Codex CLI** — в терминале (macOS/Linux) или PowerShell (Windows), вне Claude Code. Это разовая штука: plan-tango внутри дёргает именно эту установку.

```bash
npm install -g @openai/codex
codex login
```

**2. Поставь сам плагин** — это уже внутри Claude Code-сессии. Две slash-команды ниже набираешь прямо в чате с агентом, **по одной за сообщение** (Claude Code считает одно сообщение в чате одной командой — если вставить оба сразу, вторая уедет аргументом в URL первой):

```
/plugin marketplace add egsok/plan-tango
```

Дождись, пока маркетплейс добавится, и отправь второе сообщение:

```
/plugin install plan-tango@plan-tango
```

Альтернатива — попросить агента: «установи мне plan-tango из github.com/egsok/plan-tango». Claude разберётся, что делать, и поможет.

**3. Перезагрузи плагины** — в **терминальном** Claude Code: `/reload-plugins` прямо в чате, чтобы скилл, скрипты и агент подхватились без выхода из сессии. В **VS Code расширении** этой команды нет — используй **Developer: Reload Window** из Command Palette (или перезапусти Claude Code).

## Использование

```
/plan-tango:tango                          # активный план из plan mode
/plan-tango:tango <slug-or-path>           # или явный путь к файлу
/plan-tango:tango --fast                   # приоритетный режим Codex (~1.5× быстрее)
/plan-tango:tango --max-iter 10 --effort medium --lenient --quiet --fresh-each
```

Постоянные настройки лежат в `~/.claude/plan-tango/config.json`. Если не хочется редактировать JSON руками — интерактивная настройка: `/plan-tango:settings`.

Полная документация по флагам, статусам и архитектуре — [plugins/plan-tango/README.md](plugins/plan-tango/README.md) (English) · [plugins/plan-tango/README.ru.md](plugins/plan-tango/README.ru.md) (Russian).

## Что плагин делает помимо самого цикла

- **Режим тредов (continue / fresh).** По умолчанию все итерации идут в одном треде Codex (`continue`): дешевле за счёт попаданий в кеш промпта на повторяющихся блоках плана, быстрее, и в панели Codex один тред на весь прогон. Начиная со второй итерации в промпт подсыпается блок сброса — короткая инструкция «забудь свои прошлые выводы по этому плану и оценивай заново», чтобы Codex не упирался в собственный прошлый вердикт (якорный эффект). Если хочется полностью независимого разбора в каждом раунде — `--fresh-each` или `thread_mode: fresh` в настройках.
- **Умная остановка по тяжести замечаний.** Если в раунде остались только косметические замечания (форматирование, придирки к формулировкам), плагин не запускает новый раунд правок: косметика, прогнанная автоматическими правками, обычно становится только хуже. Такие замечания попадают в список рекомендаций в финальном отчёте — применяешь вручную что захочешь. Отключить — `severity_aware: false` в настройках.
- **Снимки и контроль целостности.** Перед каждым применением правок рядом с планом пишется резервная копия `.iter{N}.bak`. Перед каждой итерацией sha256 плана сверяется с `last_known`; если план поменялся снаружи цикла (сохранение в IDE, другой инструмент), плагин останавливается с понятной ошибкой и не затирает чужие правки.
- **Возобновление.** Состояние пишется после каждого применения правок; прерванный прогон (Ctrl+C, падение Claude Code, перезагрузка машины) продолжается с того же места через `--resume`.

Плюс по мелочи: файл блокировки на план (два параллельных запуска не перетирают друг друга; через 30 минут блокировка считается просроченной), защита от правок вне плана (плагин **только** правит plan-файл — если Codex предложит править код в репозитории, замечание логируется как `off_plan_blocked` и не применяется), необязательная финальная проверка через Opus по флагу `--final-check` — пригождается для планов с subagents / permissions / hooks / MCP.

## Обновление

Самый простой путь — запусти **`/plan-tango:update`**. Скилл сверится с GitHub, попросит подтверждение, сделает `git fetch + git reset --hard v<latest>` в маркетплейс-клоне и напечатает напоминание о reload. Флаг `--check` — посмотреть статус без обновления; `--force` — снести локальные модификации в маркетплейс-клоне (если ты что-то там правил руками).

Альтернативный путь (всегда работает): в Claude Code открой `/plugin → Marketplaces → plan-tango → Update`. Claude Code подтянет маркетплейс и переустановит плагин, если опубликована новая `version`. Автообновление включается там же: **Enable auto-update**. У сторонних маркетплейсов оно по умолчанию выключено — это политика Claude Code, не plan-tango.

Независимо от обоих путей plan-tango сам следит за GitHub release-каналом: на старте сессии (через `SessionStart` хук) и в конце каждого прогона он проверяет новый тег — не чаще раза в 7 дней, тихо падает на сетевых проблемах — и допечатывает одну строку, если появился новый. Отключить — `update_check: false` в `~/.claude/plan-tango/config.json`. (Опт-аут глушит автоматические уведомления, но НЕ блокирует `/plan-tango:update` — явный запуск скилла всегда сработает.)
