# Changelog (RU)

Русский перевод заметных изменений plan-tango. Полная история и более ранние версии — в [CHANGELOG.md](CHANGELOG.md). Проект следует [семантическому версионированию](https://semver.org/spec/v2.0.0.html).

## [0.7.0] — 2026-07-11

### Ломающие изменения
- **Убраны алиасы `final_check`.** Устаревшие значения конфига `auto`/`force` и CLI-флаги `--no-final-check`/`--force-final-check` теперь дают жёсткую ошибку с указанием замены (`never`/`always`, `--final-check`).

### Исправлено
- **Удалён детектор off-plan по упоминаниям файлов** — смёржен [PR #1](https://github.com/egsok/plan-tango/pull/1) от @kreml9 (спасибо!) плюс зачистка вслед. Детектор флаговал любой путь к файлу в тексте находки как «правку вне плана», хотя по контракту `location` несёт цитаты-доказательства из репо, а текст фикса законно называет файлы, *о которых* написан план. Полевые данные по реальным сессиям: 14 флагов / 14 ложных / 0 настоящих — одна сессия четырежды упёрлась в блокирующий протокол и удвоила время на рестартах. Настоящая защита всегда была структурной: `edit_plan[].file_path` равен пути плана по построению, а оркестратор строит Edit'ы только против текста плана. Статусы `off-plan-target`/`off_plan_blocked` исключены из протокола.
- **Windows-пути с обратными слэшами больше не роняют классификацию** (`stdin_not_json` срабатывал на каждом запуске) — парсер делает повторную попытку с экранированием; конвенция по-прежнему forward slashes.
- **Лояльный парсинг вердикта Codex.** Преамбула перед `ALLOW:`/`BLOCK:`, жирный маркдаун (`**BLOCK:**`), нумерация `N)` и жирные заголовки severity больше не дают MALFORMED (каждый MALFORMED стоил полного повторного ревью).
- **Стабильные хэши находок** — из нормализованных `severity :: title` вместо сырого среза прозы: детекторы stuck/oscillation переживают переформулировки одного и того же дефекта.
- **Атомарная запись `state.json`** (tmp+rename) — падение посреди записи больше не ломает `--resume`.
- **Мгновенный перехват протухших локов**, если PID владельца мёртв на том же хосте (TTL-фолбэк без изменений).
- **Очистка workspace переживает EBUSY**, когда текущая директория внутри workspace (chdir наружу + один повтор).

### Добавлено
- **`commit-iter.mjs`** — детерминированный идемпотентный пост-итерационный учёт (findings_history, хэш плана, thread-id, инкремент итерации, продление лока) вместо импровизаций оркестратора — одна реальная сессия задвоила `findings_history` и потребовала ручной починки state.
- **`evaluate-stop.mjs`** — детерминированная оценка стоп-условий вместо LLM-арифметики в шаге 21; заодно починен ложный regression-триггер после ретрая на свежем треде.
- **Pre-flight проверка worktree.** `init.mjs` сообщает, какие упомянутые в плане файлы отсутствуют в repo root; скилл предупреждает до того, как сжечь итерации против не того чекаута (реальная сессия потеряла 51 минуту).
- **Предложение Opus-финчека при max-iter** — в реальной сессии Opus поймал major-дефект, который Codex пропускал 6 итераций.

### Удалено
- Плёнка `repo_evidence_available` (константа `true` через 5 скриптов и шаблон промпта) и механизм deprecation-алиасов final_check.

## [0.6.1] — 2026-05-28

### Исправлено
- **Ошибка плагина из-за дублирования хуков.** `claude doctor` на текущем Claude Code (2.1.x) показывал `Hook load failed: Duplicate hooks file detected` для plan-tango. Поле манифеста `"hooks": "./hooks/hooks.json"` — добавленное в 0.5.1 как страховка против тогда-нестабильного авто-дискавери `hooks/hooks.json` — теперь конфликтует с автоматической загрузкой того же стандартного файла (а в новых 2.1.x добавился ещё и детект дубликатов). Убрал лишнее поле из манифеста; стандартный `hooks/hooks.json` подхватывается автоматически и грузится ровно один раз. Затрагивало все установки на текущем Claude Code; влияние ограничивалось тем, что не грузился SessionStart-хук уведомления об обновлении, плюс красная ошибка в doctor — скиллы (`/plan-tango:tango`, `:settings`, `:update`) не затронуты.

## [0.6.0] — 2026-05-28

### Ломающие изменения
- **Главный скилл переименован: `run` → `tango`.** Основная команда теперь **`/plan-tango:tango`** (была `/plan-tango:run`). Старая `/plan-tango:run` больше не резолвится — поправь скрипты, заметки и привычку набора. Папка `skills/run/` переехала в `skills/tango/`; все внутренние пути скриптов, хук проверки обновлений на SessionStart (`hooks/check-update.mjs`) и ссылка из мастера настроек (`write-config.mjs` → `load-config.mjs`) перенацелены. `/plan-tango:settings` и `/plan-tango:update` не изменились.

### Почему
Расширение Claude Code для **VS Code** прятало `/plan-tango:run` из выпадающего списка автодополнения slash-команд. Пикер расширения дедуплицирует записи по leaf-имени (часть после `:`), и `run` совпадал со встроенным скиллом Claude Code `run` («Launch and drive this project's app») — побеждал встроенный, а запись плагина отбрасывалась. Команда всё ещё работала, если набрать её целиком, но в пикере была невидимой — а именно так большинство пользователей VS Code её находят и запускают. (Терминальный CLI баг не затрагивал — он резолвит полное имя.) У `tango` нет коллизии со встроенными скиллами, слово читается одинаково на русском и английском и отражает «танец» Claude↔Codex — так что команда снова видна в дропдауне VS Code.

### Миграция
- Используй **`/plan-tango:tango`** вместо `/plan-tango:run` — позиционный аргумент и флаги те же.
- После обновления перезагрузи плагины (терминал: `/reload-plugins`; VS Code: **Developer: Reload Window** или перезапуск Claude Code).
