# Changelog (RU)

Русский перевод заметных изменений plan-tango. Полная история и более ранние версии — в [CHANGELOG.md](CHANGELOG.md). Проект следует [семантическому версионированию](https://semver.org/spec/v2.0.0.html).

## [0.6.0] — 2026-05-28

### Ломающие изменения
- **Главный скилл переименован: `run` → `tango`.** Основная команда теперь **`/plan-tango:tango`** (была `/plan-tango:run`). Старая `/plan-tango:run` больше не резолвится — поправь скрипты, заметки и привычку набора. Папка `skills/run/` переехала в `skills/tango/`; все внутренние пути скриптов, хук проверки обновлений на SessionStart (`hooks/check-update.mjs`) и ссылка из мастера настроек (`write-config.mjs` → `load-config.mjs`) перенацелены. `/plan-tango:settings` и `/plan-tango:update` не изменились.

### Почему
Расширение Claude Code для **VS Code** прятало `/plan-tango:run` из выпадающего списка автодополнения slash-команд. Пикер расширения дедуплицирует записи по leaf-имени (часть после `:`), и `run` совпадал со встроенным скиллом Claude Code `run` («Launch and drive this project's app») — побеждал встроенный, а запись плагина отбрасывалась. Команда всё ещё работала, если набрать её целиком, но в пикере была невидимой — а именно так большинство пользователей VS Code её находят и запускают. (Терминальный CLI баг не затрагивал — он резолвит полное имя.) У `tango` нет коллизии со встроенными скиллами, слово читается одинаково на русском и английском и отражает «танец» Claude↔Codex — так что команда снова видна в дропдауне VS Code.

### Миграция
- Используй **`/plan-tango:tango`** вместо `/plan-tango:run` — позиционный аргумент и флаги те же.
- После обновления перезагрузи плагины (терминал: `/reload-plugins`; VS Code: **Developer: Reload Window** или перезапуск Claude Code).
